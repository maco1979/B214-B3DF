## 重新设计功能栏为分类折叠菜单

### 1. 需求分析
- 当前功能栏是平面列表，缺乏清晰的分类层级
- 菜单项数量较多，导致视觉上不够简洁
- 用户希望将功能归类，点击进出（展开/折叠）
- 需要重新设计UI，使其更符合逻辑和用户习惯

### 2. 设计方案

#### 2.1 菜单分类
将现有功能重新组织为7个清晰的分类组：
- **核心功能**：仪表盘、AI控制
- **AI应用**：农业AI、模型管理、推理服务
- **系统管理**：系统监控、性能监控、设备自动化
- **区块链与安全**：区块链、联邦学习
- **智能体服务**：智能体管理、自动检查、调度服务、AI微调、场景管理
- **用户与社区**：用户习惯分析、社区、AI助手
- **设置**：设置

#### 2.2 组件结构设计
- 实现可折叠的菜单组组件 `SidebarGroup`
- 每个组包含标题、图标和可展开/折叠的菜单项列表
- 保持现有侧边栏的折叠/展开功能
- 优化视觉设计，增强分组的视觉区分

#### 2.3 技术实现
- 使用React状态管理菜单组的展开/折叠状态
- 使用framer-motion实现平滑的展开/折叠动画
- 保持现有路由导航和数据预取功能
- 优化移动端响应式设计

### 3. 实现步骤

#### 3.1 更新菜单数据结构
- 修改 `menuItems` 数组，将菜单项组织为分组结构
- 为每个组添加标题、图标和唯一标识符

#### 3.2 创建SidebarGroup组件
- 实现可折叠的菜单组组件
- 包含标题、图标和展开/折叠按钮
- 支持平滑的动画效果

#### 3.3 更新Sidebar组件
- 使用新的分组结构渲染菜单
- 集成SidebarGroup组件
- 优化样式和布局

#### 3.4 保持现有功能
- 保留路由导航功能
- 保留数据预取功能
- 保留侧边栏的折叠/展开功能

### 4. 预期效果
- 更清晰的功能分类，提高用户导航效率
- 更简洁的视觉设计，减少信息过载
- 可折叠的菜单组，允许用户自定义显示内容
- 平滑的动画效果，提升用户体验
- 保持与现有系统的兼容性

### 5. 文件修改
- `src/components/layout/Sidebar.tsx`：重新设计Sidebar组件，实现分类折叠菜单
- 无其他文件修改，保持现有功能完整

### 6. 技术栈
- React 函数组件
- TypeScript 类型安全
- Framer Motion 动画效果
- Tailwind CSS 样式
- React Router 路由管理

这个设计将使功能栏更加清晰、易用，符合用户的导航习惯，同时保持现有系统的功能完整性。
